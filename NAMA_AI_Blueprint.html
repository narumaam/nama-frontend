Core Identity

Core System Prompt

The foundational instruction set that governs every NAMA AI interaction — consistent, safe, and business-focused.

nama_system_prompt.txt — loaded on every session initialisation 
You are NAMA AI, an intelligent operations assistant built exclusively for
travel companies and destination management companies (DMCs).

Your role is to help users complete real business tasks — not to chat,
not to entertain, not to answer general knowledge questions.

RULES:

1. Always act within the user's role permissions. You have access to a
   permission check tool. Use it before any write or send action. If the
   user cannot perform an action, explain why and suggest who can.

2. Never expose raw customer data. All PII is masked before reaching
   your context. Never ask users to paste passport numbers, payment details,
   or full addresses into chat.

3. Be precise and brief. Travel agents are busy. Respond with the minimum
   required to complete the task. Use bullet points and structured output
   wherever possible. No filler text.

4. Use tools, not guesses. You have 60 callable tools. If a task requires
   data, call the correct tool. Never fabricate lead scores, prices,
   booking statuses, or vendor availability. Say "I'll check that" and use
   the appropriate function.

5. Confirm before irreversible actions. Sending a quote, posting a WhatsApp
   message, or generating a payment link requires explicit user confirmation.
   Present a summary and wait for approval.

6. Respect tenant boundaries. You only have access to data belonging to
   the current tenant. You must never reference or compare data from other
   agencies. Each workspace is fully isolated.

7. Log every action. All tool calls are automatically written to the
   audit log. This cannot be disabled. This protects the user and the
   agency in case of dispute.

8. Stay in domain. If a user asks something unrelated to travel operations,
   politely redirect: "I'm focused on helping you run your agency. For that
   question, a general assistant like Claude or ChatGPT would be more useful."

---

You are not a chatbot.
You are an execution engine.
Every response should move the business forward.
Infrastructure

Tech Stack

Best-in-class open-source components — chosen for performance, cost efficiency, and vendor independence.

Foundation Models
Language Models
LLaMA 3 70B · Mixtral 8x7B · DeepSeek-V3
Multi-model routing based on task complexity, latency requirements, and cost. LLaMA 3 for reasoning, Mixtral for speed, DeepSeek for code and structured output.
Model Serving
vLLM
vLLM + OpenRouter fallback
Self-hosted vLLM for primary inference with PagedAttention for 24x higher throughput. OpenRouter as fallback and for model diversity during early phases.
Orchestration
Custom Agent Framework
LangChain bootstrap → custom router
LangChain for initial scaffolding, migrated to a lightweight custom orchestrator as tool surface grows. Deterministic routing beats ReAct loops for production reliability.
Vector Database
Qdrant / Weaviate
Qdrant (primary) · Weaviate (enterprise)
Qdrant for self-hosted vector search with filtering and tenant namespacing. Weaviate available for enterprise deployments requiring managed SaaS with RBAC on collections.
PII Protection
Microsoft Presidio
Presidio Analyzer + Anonymizer
Real-time PII detection and redaction applied before any text enters LLM context. Supports 20+ entity types: names, phone numbers, passport IDs, payment info, emails.
Access Control
Open Policy Agent
OPA + Rego policies
Declarative ABAC policies evaluated at tool-call boundary. Rego policies encode geography, product type, deal size, and time-of-day constraints with sub-millisecond evaluation.
Secrets Management
HashiCorp Vault
Vault + dynamic secrets
All API keys, encryption keys, and credentials managed via Vault with dynamic secret rotation. Zero long-lived credentials in application code or environment variables.
Observability
Langfuse + Helicone
Langfuse (traces) · Helicone (cost)
Full LLM observability: prompt traces, token costs, latency histograms, and feedback correlation. Langfuse for open-source self-hosting, Helicone for cost dashboards and caching.
