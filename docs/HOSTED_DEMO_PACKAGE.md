## What To Show

- Start with the homepage or waitlist if you want to establish brand and positioning.
- Move into `/register` and show that the workspace identity is actually shaping the product.
- Use `/dashboard/team` to show real team structure, invites, and operating roles.
- Use `/dashboard/leads` to show the lead moving through the commercial workflow.
- Use `/dashboard/deals?case=maldives-honeymoon` to show the case, supplier/commercial logic, and the deal narrative.
- Use `/dashboard/finance` to show the margin and deposit control layer.
- Use `/dashboard/bookings?case=maldives-honeymoon`, `/dashboard/invoices/maldives-honeymoon`, and `/dashboard/traveler-pdf/maldives-honeymoon` to show downstream handoff and artifact visibility.
- Finish with `/dashboard/admin?entry=super-admin` to show the platform oversight layer and close the story with control.

## What To Avoid

- Do not open `autopilot`, `ekla`, or `evolution` in the main demo unless specifically asked.
- Do not make claims of live supplier or payment production rails.
- Do not linger on pages that still read as conceptual boards.
- Do not improvise outside the route map if a screen is slow; use the fallback path instead.
- Do not describe preview-safe flows as final production truth.

1. `/dashboard/demo-pack/maldives-honeymoon`
2. `/dashboard/invoices/maldives-honeymoon`
3. `/dashboard/traveler-pdf/maldives-honeymoon`
4. `/dashboard/artifacts`
5. `/dashboard/admin/audit-report`
6. `/super-admin/login`

If the session is lost or the current user is redirected, use:

- `/workspace/login`
- `/register`

## Operator Checklist

- Confirm the correct domain is live before the session starts.
- Confirm the latest branch is deployed.
- Confirm the target browser is logged into the correct environment.
- Open the homepage first, then the demo route map in order.
- Keep the primary case on `maldives-honeymoon`.
- Avoid changing data mid-demo unless the action is part of the story.
- Use the backup path immediately if a screen is slow or unclear.
- End on the control tower or audit report so the story closes with confidence.
- Keep the demo under control rather than trying to show every module.

## Application-Facing Product Summary

NAMA is an AI operating system for travel businesses. It turns workspace setup, team activation, lead capture, deal progression, finance control, booking handoff, branded traveler artifacts, and admin oversight into one connected operating flow. The current branch is strongest as a Demo and Alpha foundation, and the hosted Demo should be presented as a cohesive operating system for travel teams rather than a set of disconnected screens.
