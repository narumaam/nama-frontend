# Founder Alpha Handoff

## What Alpha Now Proves

- A travel company can register a branded workspace with plan, domain choice, and banking details.
- The same tenant identity flows through team setup, leads, finance, bookings, invoice, traveler PDF, and demo pack outputs.
- A Super Admin can enter through a separate control-tower URL and review tenant health, usage signals, audit history, and scenario seeds.
- The branch supports repeatable demo data through seeded scenarios and smoke-tested founder flows.

## What Still Feels Partial

- CRM, deals, content, comms, DMC, analytics, Autopilot, EKLA, and Evolution are still partly preview-led.
- External integrations are mostly mock, readiness, or sandbox-backed rather than production-complete.
- Super Admin access separation is real for alpha, but still uses a lightweight demo gate rather than final auth.

## What Is Not Production Ready

- Security hardening is incomplete.
- Invite delivery is not a live email system.
- Payment and supplier integrations are not yet fully live or reconciled.
- Several flows still rely on shared client-side demo state rather than backend source of truth.

## Phase 1 Starts With

1. Secure auth, roles, and tenant provisioning.
2. Durable backend-backed workflow state across the core founder journey.
3. Live test-mode payments, invite delivery, and webhook handling.
4. A truthful sandbox sourcing path across TBO, Amadeus, and Bokun.
