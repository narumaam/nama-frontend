# MVP Implementation Roadmap

## Goal

Move NAMA from a strong demo/alpha operating build into a customer-credible MVP.

## Phase 1: Demo And Alpha Completion

- eliminate demo-path dead ends
- harden weak investor-visible screens
- consolidate documentation
- maintain green build and smoke posture

## Phase 2: Core Commercial Truth

- make leads and deals durable
- move core case state to authenticated backend truth
- formalize finance and booking release state
- keep artifacts downstream of real state transitions

## Phase 3: External Rails

- truthful test-mode payments
- provider-backed invite and message delivery
- truthful sandbox sourcing path
- stronger audit trail around external actions

## Phase 4: Operational Depth

- itinerary editorial lifecycle
- DMC workflow depth
- content publishing flow
- analytics with clearer data provenance

## Phase 5: Intelligence Layer

- post-Alpha review of decision trace capture
- reason logging on commercial overrides
- structured recommendation vs human override storage
- similarity and guidance engine later

## Priority Order

1. Demo-safe continuity
2. Alpha-safe stability
3. Durable core workflow
4. External rail truth
5. Decision intelligence
